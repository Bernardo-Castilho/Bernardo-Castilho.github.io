Barbershop Simulation

System Description

We are modeling a barber shop with the following qualities:

  • The shop contains one barber and one barberâ€™s chair, open for eight hours in a day.
  • Customers arrive on average every 18 minutes, with the arrival time varying between 12 and 24 minutes.
  • If the barber is busy, the customer will wait in a queue.
  • Once the barber is free, the next customer will have a haircut.
  • Each haircut takes between 12 and 18 minutes, with the average being 15 minutes.
  • Once the haircut is done, the customer will leave the shop.

We want to answer these questions:

  1. How utilised is the barber through the day?
  2. How long does the queue get?
  3. On average, how long does a customer have to wait?

Model Description

The model was implemented as follows:

  • A BarberShop class that extends Simulation and adds two queues, one to represent the barber and one to represent the waiting area. The queues control the flow of entities through the model and collect usage statistics.
  • When the simulation starts, it sets the duration of the run and uses the generateEntities method to create customers with random inter-arrival times (18 plus or minus 6 minutes).

  • The Customer class extends Entity and has a script function that specifies the actions performed by customers:

    1. Enter the waiting queue,
    2. Seize the barber (waiting in the queue if necessary),
    3. Leave the waiting queue,
    4. Experience a delay to represent the haricut, and
    5. Release the barber.
  • When the simulation ends, it generates a report using the simulation variables and utility methods such as getStatsTable and getHistogramChart.
